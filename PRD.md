# 청년성서모임 그룹공부 공간 예약 시스템 — PRD

> 작성일: 2026-06-23 · 상태: **확정(v1.0)**

## Context (왜 만드는가)

청년성서모임은 그룹공부(약 2시간) 중심으로 운영되며, 주로 가톨릭 청소년 센터의 한정된 공간에서 여러 그룹이 모인다. 현재는 **구글폼 → 구글 캘린더** 흐름으로 예약을 받는데 다음이 불편하다.

- **중복·충돌 예약**: 같은 방·시간에 두 그룹이 겹쳐 잡힘
- **빈 시간 파악 어려움**: 어디가 비었는지 한눈에 안 보여 매번 캘린더를 뒤져야 함
- **변경·취소 번거로움**: 수정/취소가 수동이라 캘린더와 어긋남
- **알림 부재**: 확정/현황을 알기 어려움

목표: **모바일 웹 링크 하나로, 리더가 직접 빈 방·시간을 보고 충돌 없이 예약**하는 시스템을 만들어 구글폼+캘린더를 **완전히 대체**한다.

## 1. 제품 개요

- **제품명(가칭)**: 청년성서모임 공간 예약
- **한 줄 정의**: 그룹 리더가 모바일에서 빈 방과 시간을 확인하고 충돌 없이 예약·변경·취소하는 웹 서비스
- **핵심 가치**: ① 중복 예약 원천 차단 ② 빈 시간 한눈에 ③ 셀프 변경·취소

## 2. 사용자 / 페르소나

- **그룹 봉사자(리더)** — 주 사용자. 자기 그룹의 그룹공부 공간을 예약/변경/취소.
- **(참고) 운영자** — MVP 범위 밖. v2에서 관리자 화면으로 추가 예정.

## 3. 핵심 사용자 시나리오

1. 리더가 링크 접속 → 날짜 선택 → 방별 시간표(빈 칸/예약됨)를 본다.
2. 빈 시간 블록을 탭 → **이름·세례명·단계·그룹 원명수** 입력 → 예약 확정.
3. 같은 방·시간에 이미 예약이 있으면 **선택 불가/충돌 안내**.
4. 내가 만든 예약을 찾아 **시간 변경 또는 취소**.

## 4. 기능 요구사항 (MVP / v1)

### 4.1 가용 현황 보기 (최우선)
- 날짜 기준으로 **방 × 시간대(1시간 블록) 그리드** 표시.
  - 빈 칸: 예약 가능 (탭하면 예약 폼 이동)
  - 찬 칸: 봉사자 이름·세례명·단계·그룹 원명수 표시
- 모바일 우선 레이아웃. 오늘/이번 주 빠른 이동.
- 지난 시간 블록은 예약 불가(비활성) 처리.

### 4.2 예약 생성
- 빈 슬롯 선택 → 입력 폼:
  - **봉사자 이름** (텍스트)
  - **세례명** (텍스트)
  - **그룹 단계** (선택: 창세기 / 탈출기 / 마르코 / 요한 / 사도행전 / 이사야)
  - **그룹 원명수** (숫자)
  - 방·날짜·시작시간은 선택값 자동 채움, 종료 시간은 시작+2시간 기본값(수동 변경 가능)
- **충돌 검증**: 저장 시 같은 방·겹치는 시간 예약이 있으면 거부하고 사유 안내 (동시성 고려 — DB 레벨 제약).

### 4.3 예약 조회 / 변경 / 취소
- 입력한 이름+세례명으로 **내 예약 목록** 조회.
- 시간/방/원명수 변경(= 재충돌 검증), 취소.
- 취소 시 "정말 취소하시겠습니까?" 확인 단계.

### 4.4 식별
- 로그인 없음. 이름+세례명은 예약 조회·수정의 식별 키로 사용.
- 그리드 상 예약 칸에 **봉사자 이름, 세례명, 단계, 그룹 원명수** 전체 노출.

### 비범위 (v1 제외 → v2 후보)
- 관리자/운영자 화면(강제 취소, 방·휴일 관리, 통계)
- 알림(카카오/이메일), 반복(정기) 예약
- 개인 로그인 계정, 예약 정책 제한(기간/횟수)
- 구글 캘린더 동기화

## 5. 데이터 모델

### rooms
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| name | text | 방 이름 (예: 세미나실A) |
| capacity | int | 정원 (optional) |
| is_active | bool | 비활성화 가능 |

### bookings
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| room_id | uuid FK → rooms | |
| date | date | 예약 날짜 |
| start_time | time | 시작 시간 (1시간 단위) |
| end_time | time | 종료 시간 (기본 start + 2h) |
| leader_name | text | 봉사자 이름 |
| baptismal_name | text | 세례명 |
| group_stage | text | 단계 (창세기/탈출기/마르코/요한/사도행전/이사야) |
| member_count | int | 그룹 원명수 |
| created_at | timestamptz | |

**충돌 방지**: `(room_id, date, start_time, end_time)` 시간 겹침을 DB 트랜잭션 또는 Postgres exclusion constraint로 원천 차단.

## 7. 비기능 요구사항

- **모바일 웹** 최적화 (360px 이상), 빠른 로딩.
- **동시성 안전**: 두 사람이 같은 슬롯을 동시에 눌러도 한 건만 성공.
- 단일 DB 진실 소스 — 화면 새로고침 시 항상 최신 상태 반영.
- 중간 규모 트래픽(수십 그룹, 주 수십~수백 예약) 충분히 감당.

## 8. 권장 기술 스택

| 레이어 | 선택 | 이유 |
|--------|------|------|
| 프론트엔드 | **Next.js (React)** | 모바일 웹, Vercel 배포 최적화 |
| 백엔드/DB | **Supabase (Postgres)** | 충돌 방지 제약, 실시간 반영, MCP 연결됨 |
| 배포 | **Vercel** | 링크 형태 공유, MCP 연결됨 |

## 9. 성공 지표

- 구글폼+캘린더 사용 0건으로 전환 (완전 대체)
- 중복 예약 사고 0건
- 리더가 도움 없이 예약·변경·취소 완료 (이탈 없이)
